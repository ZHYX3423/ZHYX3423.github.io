入坑 Rust 的第一周

花了差不多一周的时间过了一遍 Rust 的基础语法。说实话，刚开始的体验有点痛苦——编译器像一个严格的老师，每一行代码都要审查得明明白白。

所有权机制

Rust 最核心的概念就是所有权（ownership）。没有垃圾回收，没有手动 malloc/free，而是通过一套编译期的借用检查规则来保证内存安全。

一开始很不习惯：

let s1 = String::from("hello");
let s2 = s1;             // s1 已经失效了
println!("{}", s1);      // 编译错误！

每次把变量传给函数、放进循环，都得多想一步"这个值现在归谁"。但几天下来，这种思维方式反而让人对内存管理有了更清晰的认识。

第一印象

  • 编译器信息极其友好：报错信息通常会给出修改建议，甚至有示例代码
  • Cargo 很好用：内置的包管理和构建工具，体验比 C++ 的 CMake 舒服太多
  • 文档质量高：官方的那本 The Book 写得通俗易懂

虽然离真正用 Rust 做项目还有一段距离，但至少已经不再是"Hello World 程序员"了。下一阶段打算用 Rust 写个小工具练练手。
